# PMBOK-Aligned Risk Mapping

This is a summarized PMBOK-aligned risk reference for the Project Risk Monitor MVP. It is not a copy of PMBOK text.

## Primary project impact categories

- Scope Risk
- Schedule Risk
- Cost Risk
- Quality Risk
- Resource Risk
- Stakeholder Risk
- Communication Risk
- Procurement / Vendor Risk
- Technical / Integration Risk
- Data Risk
- Compliance / Security Risk
- Adoption / Change Risk

## Classification guidance

- If work cannot proceed now, classify as Blocker.
- If impact may happen later, classify as Risk.
- If another person, team, vendor, or system must act first, classify as Dependency.
- If a decision or approval is required, classify as Pending Decision.
- If a due date has passed, classify as Overdue Action.
- If the update changes agreed scope, classify as Scope Change.
